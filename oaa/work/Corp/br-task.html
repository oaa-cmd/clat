Brown Corpus Explorer

Учебная эмуляция корпуса Brown Corpus

Brown Corpus Explorer — это учебная эмуляция, а Брауновского корпуса. Эмуляция воспроизводит основные принципы, интерфейсные элементы и типы анализа, характерные для работы с корпусом, но использует ограниченный набор демонстрационных данных.

Что именно эмулируется

Эмуляция воспроизводит следующие аспекты работы с реальным корпусом:

  • Жанровая структура — все 15 категорий с кодами, названиями и статистикой (9 категорий содержат тексты)
  • POS-разметка — каждое слово в демо-текстах снабжено тегом Brown tagset
  • Конкорданс (KWIC) — поиск слова с отображением контекста Key Word In Context
  • Частотный анализ — топ-30 самых частотных слов с реальными данными Brown Corpus
  • N-граммы — биграммы и триграммы с частотами (приближёнными к реальным)
  • Коллокации — слова, статистически связанные с искомым словом
  • Статистика — обзор корпуса, диаграммы распределения по жанрам и частям речи

Технологии создания

Приложение реализовано как одностраничное веб-приложение (SPA) с использованием исключительно клиентских технологий:

  • HTML5 — структура документа, семантическая разметка
  • CSS3 — визуальное оформление, адаптивная вёрстка, анимации, градиенты
  • Vanilla JavaScript — вся логика приложения без внешних библиотек и фреймворков

Приложение не требует сервера, базы данных или подключения к Интернету. Все данные встроены непосредственно в код. Это означает, что файл можно просто открыть в любом современном браузере.

Пояснение: SPA (Single Page Application) — архитектурный подход, при котором весь пользовательский интерфейс реализуется на одной HTML-странице. Вместо загрузки новых страниц происходит динамическое обновление содержимого с помощью JavaScript.

Структура данных в приложении

Данные в приложении организованы в виде JavaScript-объектов и массивов:

  • categories — массив из 15 объектов с информацией о жанровых категориях
  • sampleTexts — объект, ключи которого — коды жанров, значения — массивы размеченных текстов (каждое слово — пара [слово, тег])
  • topFreqWords — массив 30 самых частотных слов с реальными частотами из Brown Corpus
  • bigrams / trigrams — массивы частотных биграмм и триграмм
  • collocationsData — объект с коллокатами для нескольких ключевых слов
  • posTagDescriptions — словарь описаний POS-тегов
Пример: как хранятся размеченные тексты
{
  id: 'A01',
  title: 'Atlanta Constitution (Reportage)',
  words: [
    ['The',   'AT'],     // артикль
    ['Fulton','NP'],     // имя собственное
    ['County','NN'],     // существительное
    ['Grand', 'JJ'],     // прилагательное
    ['Jury',  'NN'],     // существительное
    ['said',  'VBD'],    // глагол (прош. вр.)
    ...
  ]
}

Функциональность эмуляции

Вкладка «Корпус» — просмотр размеченных текстов

Основная вкладка, на которой отображаются тексты выбранного жанра. Каждое слово снабжено POS-тегом в формате надстрочного индекса (суперскрипт). Функции:

  • Переключение жанров — клик по категории на боковой панели
  • Клик по слову — всплывающая карточка с информацией: само слово, POS-тег с описанием, частотность в демо-текстах и в полном корпусе (если слово входит в топ-30)
  • Показать/скрыть теги — кнопка для переключения отображения POS-тегов

Вкладка «Конкорданс (KWIC)»

KWIC (Key Word In Context) — один из фундаментальных инструментов корпусной лингвистики. Формат отображения, при котором искомое слово показывается по центру с окружающим контекстом слева и справа. Для использования:

  1. Введите слово в поисковую строку
  2. Нажмите кнопку «Concordance»
  3. Результаты отображаются в виде таблицы KWIC

Таблица содержит столбцы: левый контекст, ключевое слово, правый контекст, POS-тег и идентификатор источника.

Совет: Попробуйте искать слова: the, he, she, was, said, government, time. Эти слова присутствуют в демо-текстах и дадут результаты.

Вкладка «Частотность»

Отображает топ-30 самых частотных слов в Брауновском корпусе. Данные этого раздела — реальные (основаны на опубликованных частотных списках Кучеры и Фрэнсиса). Для каждого слова указаны:

  • Ранг (позиция в частотном списке)
  • Само слово
  • Визуальная полоска, отражающая относительную частотность
  • Основной POS-тег
  • Абсолютная частота (количество вхождений)

Сверху отображается столбчатая диаграмма для первых 20 слов.

Вкладка «N-граммы»

Показывает наиболее частотные последовательности из 2 слов (биграммы) и из 3 слов (триграммы). Можно фильтровать по типу (биграммы/триграммы) и по жанру.

Пояснение: N-граммы помогают выявлять устойчивые словосочетания, синтаксические модели и языковые паттерны. Например, биграмма «of the» — самая частотная в Brown Corpus (~9781 вхождений), что отражает высокую частотность предложной конструкции «X of the Y» в английском языке.

Функция «Collocations» (коллокации)

При вводе слова и нажатии кнопки «Collocations» отображается облако коллокатов — слов, которые статистически значимо часто встречаются рядом с данным словом. Для каждого коллоката указан условный показатель MI (Mutual Information).

Коллокации доступны для слов: time, make, government, man, world, people, school, the.

Вкладка «Статистика»

Обзорный раздел с количественной информацией о корпусе:

  • Карточки с основными цифрами (тексты, словоупотребления, словоформы, жанры, теги, год)
  • Диаграмма распределения текстов по жанрам
  • Диаграмма распределения частей речи

Ограничения эмуляции

Принципиальные ограничения

Осознание ограничений эмуляции — важная часть обучения. Вот что следует учитывать:

АспектРеальный Brown CorpusУчебная эмуляция
Объём ~1 014 000 словоупотреблений ~800 словоупотреблений
Тексты 500 реальных текстовых фрагментов ~12 демонстрационных фрагментов
Жанры с текстами 15 (все категории) 9 категорий содержат тексты
Конкорданс Полнотекстовый поиск по всему корпусу Поиск только по демо-текстам
Частотные данные Точные подсчёты по всему корпусу Реальные данные для топ-30; остальное — демо

Тексты Аутентичные тексты 1961 года Стилизованные образцы (кроме A01)

Что НЕ эмулируется

  • Полнотекстовый поиск — нельзя найти произвольное слово (только те, что есть в демо-текстах)
  • Регулярные выражения — в настоящих корпусных инструментах (AntConc, CQPweb) можно искать по шаблонам
  • Сортировка конкорданса — в реальных инструментах KWIC можно сортировать по левому или правому контексту
  • Подкорпуса — нельзя создать собственный подкорпус и сравнить его с другим
  • Статистические тесты — log-likelihood, chi-square и другие меры не реализованы
  • Леммантизация — нет приведения словоформ к лемме (например, goes, went, going → go)
Важно: Эмуляция создана для ознакомления с принципами корпусной лингвистики, а не для замены реальных исследовательских инструментов. После освоения эмуляции рекомендуется перейти к работе с настоящими корпусными менеджерами: AntConc (бесплатный, оффлайн), Sketch Engine (онлайн, платный), COCA (corpus.byu.edu, онлайн).

Инструкция по работе с эмуляцией

Запуск приложения

  1. Сохраните файл эмуляции с расширением .html (например, brown-corpus-explorer.html)
  2. Откройте файл двойным щелчком — он запустится в вашем браузере по умолчанию
  3. Рекомендуемые браузеры: Chrome, Firefox, Edge (любая современная версия)
  4. Подключение к Интернету не требуется
Совет: Для лучшего опыта используйте экран шириной не менее 1024 пикселей. На мобильных устройствах боковая панель скрывается, но основная функциональность доступна.

Навигация по интерфейсу

Интерфейс состоит из четырёх основных зон:

  • Шапка (Header) — название приложения и краткая статистика корпуса
  • Боковая панель (Sidebar) — список жанровых категорий и справочник POS-тегов
  • Панель инструментов (Toolbar) — поисковая строка и кнопки действий
  • Основная область (Content) — вкладки с содержимым (Корпус, Конкорданс, Частотность, N-граммы, Статистика)

Пошаговый сценарий работы

Сценарий 1: Изучение размеченного текста

  1. На боковой панели выберите жанр (например, A — Press: Reportage)
  2. Прочитайте текст, обращая внимание на POS-теги над словами
  3. Кликните на любое слово — появится карточка с подробной информацией
  4. Попробуйте скрыть теги (кнопка «Скрыть теги»), затем снова показать
  5. Переключитесь на другой жанр (например, K — Fiction) и сравните тексты

Сценарий 2: Поиск слова в конкордансе

  1. Введите слово the в поисковую строку
  2. Нажмите кнопку «Concordance»
  3. Изучите таблицу KWIC — обратите внимание на разнообразие контекстов
  4. Попробуйте другие слова: said, was, she

Сценарий 3: Анализ частотности

  1. Перейдите на вкладку «Частотность»
  2. Изучите столбчатую диаграмму — какие слова самые частотные?
  3. Обратите внимание на POS-теги — какие части речи доминируют?
  4. Подумайте, почему именно эти слова входят в топ

Сценарий 4: Исследование коллокаций

  1. Введите слово time в поисковую строку
  2. Нажмите кнопку «Collocations»
  3. Изучите облако коллокатов и таблицу с MI-scores
  4. Повторите для слов make и government

Горячие клавиши и подсказки

  • Enter в поисковой строке — запускает поиск конкорданса
  • Клик по слову в тексте — показывает информацию о слове
  • Клик в любое место — закрывает всплывающую карточку слова
Примечание: Если при поиске в конкордансе слово не найдено, это не означает, что его нет в Brown Corpus — это означает лишь, что его нет среди ~800 слов демо-текстов. Приложение предложит альтернативные слова для поиска.

Практические задания

Задания по работе с эмуляцией

Исследование POS-разметки

Откройте текст A01 (Press: Reportage) и выполните следующее:

  1. Найдите все существительные (теги NN, NNS) в тексте. Сколько их?
  2. Найдите все глаголы в прошедшем времени (VBD). Выпишите их
  3. Какой POS-тег имеет слово Friday? Почему это не NP (имя собственное)?
  4. Сравните с текстом K01 (Fiction). Каких тегов больше в художественном тексте?
Используйте кнопку «Показать/Скрыть теги» для самопроверки: сначала попробуйте определить теги самостоятельно, затем включите отображение.

Конкорданс и контекст

  1. Найдите в конкордансе слово the. Сколько вхождений найдено?
  2. Изучите правый контекст — какие слова чаще всего следуют за the?
  3. Теперь найдите слово was. Изучите левый контекст — какие подлежащие стоят перед was?
  4. Сформулируйте наблюдение: какие паттерны вы заметили?
В реальном корпусном исследовании KWIC-конкорданс позволяет замечать типичные модели, которые невозможно обнаружить при чтении отдельных текстов.

Закон Ципфа на практике

  1. Откройте вкладку «Частотность» и изучите топ-30 слов
  2. Вычислите соотношение: частота слова #1 (the) делённая на частоту слова #2 (of). Запишите результат
  3. Повторите для пар: #1/#3, #1/#5, #1/#10, #1/#20
  4. Постройте простой график (на бумаге или в Excel): ось X — ранг, ось Y — частота
  5. Какую форму имеет кривая? Соответствует ли она закону Ципфа (частота ≈ 1/ранг)?
Закон Ципфа утверждает, что частота слова обратно пропорциональна его рангу в частотном списке. Это универсальная закономерность, наблюдаемая в любом достаточно большом тексте.

Жанровые различия

  1. Прочитайте тексты жанров A (Press), J (Learned) и K (Fiction)
  2. Для каждого жанра подсчитайте приблизительный процент: существительных, глаголов, прилагательных
  3. В каком жанре больше прилагательных (JJ)? Почему?
  4. В каком жанре больше местоимений (PPS, PPSS)? Почему?
  5. Какие выводы можно сделать для преподавания ESP/EAP?
Художественная проза обычно содержит больше местоимений и глаголов в прошедшем времени, а научные тексты — больше номинализаций и пассивных конструкций.

N-граммы и устойчивые сочетания

  1. Откройте вкладку «N-граммы» и изучите биграммы
  2. Выпишите пять самых частотных биграмм. Что их объединяет структурно?
  3. Переключитесь на триграммы. Найдите триграмму «in order to» — к какому жанру она отнесена?
  4. Найдите триграммы, характерные для художественных текстов (Fiction). Чем они отличаются от академических?
  5. Как эта информация может быть полезна при составлении учебных материалов?

Коллокации и лексическое окружение

  1. Найдите коллокации для слова time. Выпишите 5 самых сильных коллокатов
  2. Сделайте то же для слова make
  3. Для каждого коллоката составьте пример предложения
  4. Подумайте: какие из этих словосочетаний вы бы включили в учебник уровня B1? Уровня C1?
  5. Сравните коллокации people и man. Какие различия вы замечаете?
Коллокации — один из самых полезных результатов корпусного анализа для преподавания: они показывают, какие слова «притягиваются» друг к другу в реальном языке.

Задания с использованием LLM

Зачем использовать LLM вместе с корпусной эмуляцией

Большие языковые модели (LLM), такие как ChatGPT, Claude, Gemini и другие, могут выступать в нескольких ролях при работе с корпусными данными. Это позволяет существенно расширить учебный потенциал эмуляции:

  • LLM как эксперт — объясняет теоретические понятия, комментирует результаты, приводит дополнительные примеры
  • LLM как контролёр — проверяет ваши ответы на задания, указывает на ошибки, оценивает рассуждения
  • LLM как консультант — помогает интерпретировать данные, предлагает методические решения, отвечает на вопросы
Пояснение: LLM не заменяет корпусные данные — модель генерирует текст на основе вероятностных паттернов, а не подсчитывает реальные частоты в конкретном корпусе. Однако LLM прекрасно подходит для обсуждения, объяснения и расширения результатов, полученных из корпуса или его эмуляции.

LLM как эксперт

В этой роли LLM выступает как знающий собеседник, способный объяснить сложные понятия корпусной лингвистики, прокомментировать найденные данные и расширить ваше понимание.

LLM = Эксперт

Разбор POS-тегов с экспертом

  1. В эмуляции откройте текст A01 и скопируйте первое предложение с тегами
  2. Отправьте его в LLM с промптом (см. ниже)
  3. Обсудите ответ LLM — всё ли вам понятно? Задайте уточняющие вопросы
  4. Попросите LLM объяснить, чем Brown tagset отличается от Penn Treebank tagset
Я изучаю корпусную лингвистику и работаю с учебной эмуляцией Brown Corpus. Вот первое предложение из текста A01 с POS-тегами Brown tagset: «The/AT Fulton/NP County/NN Grand/JJ Jury/NN said/VBD Friday/NR an/AT investigation/NN of/IN Atlanta's/NP$ recent/JJ primary/NN election/NN produced/VBD no/AT evidence/NN that/CS any/DTI irregularities/NNS took/VBD place/NN.» Объясни каждый тег подробно. Почему слово «Friday» имеет тег NR, а не NP? Почему «that» получило тег CS, а не WPS?

Закон Ципфа: от данных к теории

  1. В эмуляции откройте вкладку «Частотность» и выпишите топ-10 слов с частотами
  2. Отправьте эти данные в LLM и попросите объяснить закон Ципфа
  3. Попросите LLM объяснить, почему функциональные слова (артикли, предлоги) частотнее содержательных
  4. Спросите, какие практические выводы можно сделать для преподавания английского
Я получила эти данные из учебной эмуляции Brown Corpus: the — 69971, of — 36411, and — 28852, to — 26149, a — 23237, in — 21341, that — 10594, is — 10099, was — 9816, he — 9543. Объясни мне как эксперт: 1) Как эти данные соотносятся с законом Ципфа? 2) Почему в топ-10 почти нет существительных и знаменательных глаголов? 3) Какие выводы для преподавания английского языка следуют из этого распределения?

LLM как контролёр

В этой роли LLM проверяет ваши выполненные задания, указывает на ошибки, помогает оценить качество ваших рассуждений.

LLM = Контролёр

Проверка самостоятельной разметки

  1. Возьмите следующее предложение и попытайтесь самостоятельно присвоить POS-теги Brown tagset каждому слову: «The young teacher carefully explained the new grammar rules to her students.»
  2. Запишите свой вариант разметки
  3. Отправьте его в LLM для проверки
  4. Обсудите ошибки, если они есть

LLM как консультант

В этой роли LLM помогает интерпретировать данные, предлагает методические идеи и отвечает на вопросы, возникающие в процессе работы.

LLM = Консультант

Методическое применение коллокаций

  1. В эмуляции найдите коллокации для слова make
  2. Обратитесь к LLM за консультацией по методическому применению
  3. На основе рекомендаций LLM разработайте мини-упражнение для студентов уровня B1
Я будущий преподаватель английского языка. Из корпусных данных Brown Corpus я получила следующие коллокации слова «make»: decision, money, sense, sure, clear, way, possible, effort, use, difference, point, mistake. Консультируй меня: 1) Как использовать эти данные для создания упражнения уровня B1? 2) Какие коллокации наиболее важны для этого уровня? 3) Предложи 2–3 типа упражнений на основе этих коллокаций. 4) Какие ошибки типичны для русскоязычных учащихся при использовании «make + noun»?

Интерпретация частотных данных для ESP

  1. Посмотрите в эмуляции на жанр J — Learned (Academic)
  2. Изучите триграммы, помеченные жанром J: in order to, the fact that, a number of
  3. Обратитесь к LLM как к консультанту по академическому английскому
Из учебной эмуляции Brown Corpus я узнала, что следующие триграммы характерны для академического жанра (Learned, категория J): «in order to» (178), «the fact that» (163), «a number of» (126). Консультируй меня как специалист по EAP (English for Academic Purposes): 1) Почему именно эти конструкции типичны для академического дискурса? 2) На каком уровне обучения их следует вводить? 3) Какие ещё конструкции ты ожидал бы увидеть в академических текстах? 4) Как эти данные помогут мне при подготовке студентов к IELTS Writing Task 2?

Комбинированные задания: эмуляция + LLM

Исследовательский мини-проект

Это задание объединяет работу с эмуляцией и все три роли LLM.

  1. Выбор темы: Выберите одно из слов, доступных для анализа коллокаций (time, make, government, man, world, people, school)
  2. Сбор данных (эмуляция): Найдите это слово в конкордансе, изучите его коллокации, определите его частотность и POS-тег
  3. Консультация (LLM-консультант): Спросите LLM, какие дополнительные данные о вашем слове можно получить из реального Brown Corpus
  4. Анализ: Напишите мини-отчёт (200–300 слов) о лексическом профиле выбранного слова
  5. Проверка (LLM-контролёр): Отправьте отчёт LLM для проверки и критической оценки
  6. Дополнение (LLM-эксперт): Попросите LLM дополнить ваш отчёт экспертным комментарием о роли этого слова в английском языке
  7. Методика: На основе собранных данных предложите 1 упражнение для учащихся

Критический анализ эмуляции

  1. Проработайте все вкладки эмуляции, делая заметки
  2. Напишите критический анализ (300–400 слов): что эмуляция демонстрирует хорошо, а что — нет? Какие функции отсутствуют? Какие данные неточны?
  3. Отправьте анализ LLM (роль: контролёр-эксперт) и попросите оценить полноту ваших замечаний
  4. Предложите три конкретных улучшения для эмуляции

Совет по работе с LLM: Всегда указывайте контекст в промпте: что вы изучаете, какой ваш уровень, какую роль должна играть LLM. Чем конкретнее промпт, тем полезнее ответ. Не стесняйтесь переспрашивать и просить уточнения — это естественная часть учебного диалога.
Важно: LLM может ошибаться в конкретных числовых данных (частотах, количестве вхождений). Всегда сверяйте количественные данные с корпусными инструментами. LLM наиболее полезна для объяснений, интерпретаций и методических рекомендаций.

Глоссарий

Глоссарий терминов, используемых при работе с учебным приложением, эмулирующим Брауновский корпус.

Термин (англ.) Определение (рус.)
Corpus / Корпус Систематически собранная коллекция текстов (обычно в электронной форме), предназначенная для лингвистического исследования. Может быть общим (general) или специализированным (specialized).
Token / Токен (словоупотребление) Каждое отдельное вхождение слова в тексте. В предложении «the cat sat on the mat» — 6 токенов (слово «the» считается дважды).
Type / Тип (словоформа) Уникальная форма слова. В том же предложении «the cat sat on the mat» — 5 типов (слово «the» считается один раз).
TTR (Type-Token Ratio) Отношение количества уникальных словоформ к общему числу словоупотреблений. Мера лексического разнообразия текста.
Concordance / Конкорданс Список всех вхождений заданного слова (или выражения) в корпусе с окружающим контекстом. Основной инструмент корпусного анализа.
KWIC (Key Word In Context) Формат отображения конкорданса, при котором искомое слово центрировано, а контекст расположен слева и справа.
POS tag / Частеречный тег Метка, указывающая часть речи и грамматические характеристики слова (например, NN — существительное единственного числа, VBD — глагол в прошедшем времени).
Tagset / Набор тегов Система тегов, используемая для разметки корпуса. Brown tagset содержит 87 тегов; Penn Treebank tagset — 36.
Collocation / Коллокация Статистически значимое совместное появление двух или более слов. Выявляется с помощью мер ассоциации (MI, t-score, log-likelihood).
N-gram / N-грамма Последовательность из N смежных элементов (слов). Биграмма = 2 слова, триграмма = 3 слова. Используется для выявления устойчивых сочетаний.
Frequency list / Частотный список Упорядоченный список слов корпуса с указанием их частоты (количества вхождений). Основа для определения «ядерной» лексики.
Zipf's law / Закон Ципфа Эмпирическая закономерность: частота слова обратно пропорциональна его рангу в частотном списке. Второе по частотности слово встречается примерно вдвое реже первого.
MI (Mutual Information) Мера силы ассоциации между словами. Высокий MI указывает на сильную коллокационную связь. Отдаёт предпочтение редким, но устойчивым сочетаниям.
Lemma / Лемма Каноническая (словарная) форма слова. Все формы go, goes, went, going, gone сводятся к лемме go.
Annotation / Аннотация (разметка) Дополнительная лингвистическая информация, добавленная к тексту корпуса: POS-теги, синтаксический разбор, семантические роли и т. д.
DDL (Data-Driven Learning) Подход к изучению языка, при котором учащиеся самостоятельно анализируют корпусные данные для обнаружения языковых закономерностей. Предложен Тимом Джонсом (1991).
Concordancer / Конкордансер Программа для поиска и отображения конкордансов. Примеры: AntConc (десктоп, бесплатный), Sketch Engine (онлайн, платный).
Balanced corpus / Сбалансированный корпус Корпус, в котором различные жанры и типы текстов представлены в определённых, продуманных пропорциях. Brown Corpus — классический пример сбалансированного корпуса.
Representative corpus / Репрезентативный корпус (Определение не предоставлено в исходном глоссарии.)
Примечание: Глоссарий включает термины, необходимые для работы с учебной эмуляцией.
