Web Storage API

О приложении

Это учебное приложение поможет на практике освоить работу с Web Storage API (localStorage и sessionStorage) — технологиями, которые в веб-приложениях учебного курса и позволяют хранить в браузере пользовательские данные.

Выполняя задания в DevTools, можно (!) научиться:

  • Сохранять данные в localStorage и sessionStorage
  • Получать доступ к сохраненным данным
  • Удалять данные из хранилища
  • Работать с DevTools для отладки и мониторинга хранилищ
  • Понимать разницу между localStorage и sessionStorage
  • Освоить полезные инструменты контроля безопасности своих данных

Задания по Web Storage API

Здравствуйте, Студент! Выполните следующие задания (в DevTools!) для изучения Web Storage API.

Задание 1: Знакомство с localStorage

Не выполнено

localStorage позволяет хранить данные без ограничения времени. Данные сохраняются даже после закрытия браузера.

Инструкция:

  1. Откройте DevTools в Chrome (F12 или Ctrl+Shift+I)
  2. Перейдите на вкладку "Application"
  3. В левой панели найдите раздел "Storage" и выберите "Local Storage"
  4. Нажмите кнопку ниже, чтобы сохранить ваши данные в localStorage
localStorage.setItem('studentLastName', 'ВашаФамилия');
localStorage.setItem('language', 'русский');
localStorage.setItem('course', 'Web Storage API');

Задание 2: Получение данных из localStorage

Не выполнено

Теперь научимся получать данные из localStorage.

Инструкция:

  1. В DevTools на вкладке "Application" найдите сохраненные ранее данные
  2. Нажмите кнопку ниже, чтобы получить данные из localStorage
  3. Обратите внимание на консоль DevTools (вкладка "Console")
const lastName = localStorage.getItem('studentLastName');
const language = localStorage.getItem('language');
const course = localStorage.getItem('course');
console.log(`Студент: ${lastName}, Язык: ${language}, Курс: ${course}`);

Задание 3: Работа с sessionStorage

Не выполнено

sessionStorage хранит данные только на время сессии (пока открыт браузер). При закрытии вкладки данные удаляются.

Инструкция:

  1. В DevTools на вкладке "Application" найдите раздел "Session Storage"
  2. Нажмите кнопку ниже, чтобы сохранить данные в sessionStorage
  3. Сравните, как выглядят данные в localStorage и sessionStorage
sessionStorage.setItem('currentTask', 'Задание 3');
sessionStorage.setItem('sessionStartTime', new Date().toLocaleTimeString());
sessionStorage.setItem('studentProgress', '60%');

Задание 4: Удаление данных из хранилища

Не выполнено

Теперь научимся удалять данные из хранилища.

Инструкция:

  1. В DevTools проверьте содержимое localStorage и sessionStorage
  2. Нажмите кнопку ниже, чтобы удалить один элемент из localStorage
  3. Нажмите вторую кнопку, чтобы полностью очистить sessionStorage
  4. Проверьте изменения в DevTools
// Удаление одного элемента
localStorage.removeItem('language');

// Полная очистка хранилища
sessionStorage.clear();

Задание 5: Очистка хранилищ в настройках браузера

Не выполнено

Помимо программного удаления данных, пользователь может очистить хранилища через настройки браузера.

Инструкция для Chrome:

  1. Откройте меню Chrome (три точки в правом верхнем углу)
  2. Выберите "Настройки"
  3. Перейдите в раздел "Конфиденциальность и безопасность"
  4. Выберите "Удаление истории просмотров"
  5. Выберите "Файлы cookie и другие данные сайтов"
  6. Нажмите "Удалить данные"

После выполнения этих шагов, вернитесь в DevTools и проверьте, что данные были удалены.

Альтернативный способ (быстрее):

  1. В DevTools на вкладке "Application"
  2. В левой панели найдите раздел "Storage"
  3. Нажмите правой кнопкой мыши на "Local Storage" или "Session Storage"
  4. Выберите "Clear"

Задание 7: Ограничения Web Storage API

Не выполнено

Важно понимать ограничения Web Storage API:

  • Объем хранилища ограничен (обычно 5-10 МБ)
  • Хранить можно только строки (для объектов нужно использовать JSON)
  • Нет встроенной поддержки шифрования
  • Работает синхронно, что может замедлить работу приложения

Задание:

Попробуйте сохранить объект в localStorage без преобразования в JSON, а затем с преобразованием. Сравните результаты.

// Без JSON
const linguistData = { name: 'ВашаФамилия', specialization: 'Компьютерная лингвистика' };
localStorage.setItem('linguistDataRaw', linguistData);

// С JSON
localStorage.setItem('linguistDataJSON', JSON.stringify(linguistData));

// Получение данных
console.log('Без JSON:', localStorage.getItem('linguistDataRaw'));
console.log('С JSON:', JSON.parse(localStorage.getItem('linguistDataJSON')));
