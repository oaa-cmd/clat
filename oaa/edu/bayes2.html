Байесовский пересчёт в экзаменационной ситуации

Контекст: студент и неподготовленные вопросы

Ситуация экзамена

Студент готовится к экзамену, на котором будет задано 2 вопроса из общего пула в 40. Он не успел подготовить 5 из них.

За день до экзамена преподаватель сообщает, что один из вопросов точно будет из темы «Вероятностный подход в лингвистике». Студент знает, что 2 из 5 неподготовленных вопросов относятся к этой теме, а всего в теме — 10 вопросов.

Цель студента:

  • Оценить вероятность, что среди двух экзаменационных вопросов окажется хотя бы один неподготовленный.
  • Учесть новую информацию о тематике одного из вопросов.

Пояснение: Это классический случай, когда необходимо обновить априорную оценку вероятности с учётом новой информации — то есть применить теорему Байеса.

Формализация задачи

Обозначения и вероятности

Обозначим события:

  • H: вопрос из темы «Вероятностный подход» — и он неподготовлен
  • E: известно, что вопрос из темы «Вероятностный подход»

Исходные данные:

  • Всего вопросов: 40
  • Неподготовленных: 5 (из них 2 — по вероятностному подходу)
  • Всего вопросов по вероятностному подходу: 10

Применим формулу Байеса:

𝑃 ( 𝐻 ∣ 𝐸 ) = 𝑃 ( 𝐸 ∣ 𝐻 ) ⋅ 𝑃 ( 𝐻 ) 𝑃 ( 𝐸 )

Подставим значения:

  • 𝑃 ( 𝐻 ) = 2/ 40
  • 𝑃 ( 𝐸 ∣ 𝐻 ) = 1 (если вопрос из неподготовленных по вероятностному подходу, то он точно из этой темы)
  • 𝑃 ( 𝐸 ) = 10/ 40

Результат:

𝑃 ( 𝐻 ∣ 𝐸 ) = (1*2/40) / (10/40) = 2/10 = 0.2

Пояснение: Вероятность того, что вопрос из темы «Вероятностный подход» окажется неподготовленным — 20%.

Обновлённая оценка общей вероятности

Комбинирование с другим вопросом

Один вопрос уже тематически определён — «Вероятностный подход». Второй вопрос выбирается случайно из оставшихся 39.

Вероятность, что второй вопрос окажется неподготовленным:

𝑃 ( второй неподготовлен ) = 3/ 39

Итоговая вероятность, что хотя бы один из двух вопросов окажется неподготовленным:

  • Вероятность, что оба подготовлены: 𝑃 = ( 1 − 0.2 ) ⋅ ( 1 − 3/ 39 ) ≈ 0.8 ⋅ 0.923 ≈ 0.738
  • Значит, вероятность, что хотя бы один — неподготовлен: 1 − 0.738 = 0.262

Пояснение: С учётом новой информации, риск «попасть» на неподготовленный вопрос вырос с интуитивных 5/40 = 12.5% до 26.2%.

СценарийВероятность хотя бы одного неподготовленного вопроса
Без учёта тематики≈ 12.5%
С учётом тематики (Байес)≈ 26.2%
Примечание: Байесовский пересчёт позволяет учитывать контекст и уточнения, которые интуитивная оценка игнорирует. Теорема Байеса — это инструмент для обновления знаний, а не просто формула.
Совет: Учитесь мыслить контекстуально и ответственно готовьте ВСЕ вопросы к экзамену.

Пример применения Теоремы Байеса № 1

Классификация спама в текстах

Постановка задачи

Необходимо определить, является ли входящее письмо спамом или обычным сообщением (ham) на основе содержащихся в нём слов.

Формализация: P(spam|слова) = P(слова|spam) × P(spam) / P(слова)

Компоненты модели

Априорная вероятность: P(spam) — доля спама в обучающей выборке. Например, если из 1000 писем 300 были спамом, то P(spam) = 0.3, P(ham) = 0.7.

Правдоподобие: P(слово|spam) — вероятность встретить конкретное слово в спам-письмах. Рассчитывается по частоте слова в спам-корпусе.

Расчёт для множества слов: P(w₁, w₂, ..., wₙ|spam) ≈ ∏ᵢ P(wᵢ|spam)

Пример расчёта

Письмо содержит слова: "бесплатно", "виагра", "предложение"

Данные из обучающей выборки:

  • P(spam) = 0.3, P(ham) = 0.7
  • P("бесплатно"|spam) = 0.15, P("бесплатно"|ham) = 0.01
  • P("виагра"|spam) = 0.08, P("виагра"|ham) = 0.0001
  • P("предложение"|spam) = 0.05, P("предложение"|ham) = 0.02

Вычисление:

P(spam|слова) ∝ 0.15 × 0.08 × 0.05 × 0.3 = 0.00018

P(ham|слова) ∝ 0.01 × 0.0001 × 0.02 × 0.7 = 0.000000014

Результат: P(spam|слова) >> P(ham|слова), письмо классифицируется как спам.

Важно: Для избежания проблемы нулевых вероятностей используется сглаживание Лапласа: P(слово|класс) = (count + 1) / (total + vocabulary_size).

Пример применения Теоремы Байеса № 2

Распознавание языка или диалекта

Формулировка задачи

Дан текст неизвестного языка. Необходимо определить, на каком языке он написан, используя статистические характеристики языков.

Байесовская формула: P(язык|текст) = P(текст|язык) × P(язык) / P(текст)

Признаки для классификации

Уровень символов:

  • Частоты букв (e часто в английском, é во французском)
  • N-граммы символов (th, ch в английском; qu, ou во французском)
  • Специфические символы (ñ в испанском, ß в немецком)

Уровень слов:

  • Частотные слова (the, a, is для английского; le, la, de для французского)
  • Средняя длина слов
  • Морфологические паттерны

Практический пример

Текст: "C'est une belle journée"

Анализ триграмм символов:

  • "c'e", "'es", "est" — типичны для французского
  • Апостроф между буквами — характерен для французского
  • "ée" в конце слова — французский паттерн

Расчёт вероятностей:

P(французский|признаки) = 0.89

P(английский|признаки) = 0.02

P(испанский|признаки) = 0.06

P(итальянский|признаки) = 0.03

Язык Характерные биграммы Частотные слова
Английский th, he, in, er, an the, of, and, to, a
Французский es, le, de, re, nt le, de, un, la, et
Немецкий en, er, ch, de, ei der, die, und, in, das

Пример применения Теоремы Байеса № 3

Автоматическая коррекция опечаток

Модель ошибок набора

Задача: пользователь набрал "thier", нужно предложить правильный вариант.

Формула: P(слово|набранный_текст) = P(набранный_текст|слово) × P(слово) / P(набранный_текст)

Ищем слово с максимальной апостериорной вероятностью: argmax P(слово|"thier")

Компоненты модели коррекции

P(слово) — Языковая модель: Частотность слов в корпусе. Например, P("their") = 0.003, P("tier") = 0.0001.

P(набранный_текст|слово) — Модель ошибок: Вероятность сделать конкретную опечатку. Учитывает:

  • Расстояние между клавишами (e↔i вероятнее, чем e↔z)
  • Типичные ошибки (ie↔ei очень частая ошибка)
  • Пропуски и вставки букв
  • Фонетическое сходство

Пример вычисления

Для опечатки "thier" рассматриваем кандидатов:

  • "their": P("their") = 0.003, P("thier"|"their") = 0.02 (транспозиция ei→ie)
  • "tier": P("tier") = 0.0001, P("thier"|"tier") = 0.01 (вставка h)
  • "there": P("there") = 0.002, P("thier"|"there") = 0.001 (замена re→ie)

Расчёт scores:

Score("their") = 0.003 × 0.02 = 0.00006

Score("tier") = 0.0001 × 0.01 = 0.000001

Score("there") = 0.002 × 0.001 = 0.000002

Результат: "their" имеет наивысший score и предлагается как исправление.

Пояснение: Современные системы также учитывают контекст: P(слово|контекст, набранный_текст), что значительно улучшает качество исправлений.
Важно: Байесовские методы не идеальны и имеют немало ограничений, но остаются важным инструментом благодаря своей интерпретируемости и эффективности.
